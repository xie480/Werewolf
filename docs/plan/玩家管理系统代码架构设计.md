# 玩家管理系统代码架构设计

## 1. 概述
玩家管理系统负责管理 AI 玩家的生命周期、对局内状态以及跨对局的统计数据。系统采用分层架构，结合 PostgreSQL 进行持久化存储，并利用 Redis 进行对局内状态的高性能缓存。

## 2. 核心模块与文件结构

### 2.1 数据库模型 (ORM)
**文件路径**: `ai_werewolf_core/db/models.py`
- `AIPlayerProfile`: 存储 AI 玩家的静态配置（如名称、模型提供商、模型名称、系统提示词、温度等）。
- `AIPlayerStats`: 存储 AI 玩家的跨局统计数据（如总局数、胜率、失败次数、平均耗时等）。
- `PlayerRecord`: 记录单局游戏中玩家的状态（座位号、角色、存活状态），通过 `ai_profile_id` 外键关联到 `AIPlayerProfile`。

### 2.2 数据校验与传输对象 (Schemas)
**文件路径**: `ai_werewolf_core/schemas/models.py` & `ai_werewolf_core/schemas/api.py`
- `Player` (Pydantic Model): 包含 `ai_profile_id`，用于在业务逻辑中传递玩家信息。
- API 请求/响应模型: 用于处理前端创建玩家、查询玩家列表等请求。

### 2.3 核心业务逻辑 (Core Engine)
**文件路径**: `ai_werewolf_core/core/engine/player_manager.py`
- `PlayerStatusManager`: 负责对局内玩家状态的管理。
  - **Write-Through 缓存策略**: 玩家的存活状态、身份信息等高频访问数据存储在 Redis 中（使用 Hash 和 Bitmap），同时异步同步到 PostgreSQL 的 `PlayerRecord` 表中。
  - 提供 `is_alive`, `mark_dead`, `get_player_info` 等 O(1) 复杂度的高效查询接口。

### 2.4 API 路由 (API Routes)
**文件路径**: `ai_werewolf_core/api/routes/players.py` & `ai_werewolf_core/api/routes/games.py`
- `players.py`: 提供 AI 玩家的 CRUD 接口（获取列表、创建动态玩家、查询统计数据等）。
- `games.py`: 在 `create_game` 接口中，解析玩家配置，支持从现有 AI 玩家库中选择或动态生成 AI 玩家，并将其分配到对局的 `PlayerRecord` 中。

### 2.5 Agent 工作流集成 (LangGraph)
**文件路径**: `ai_werewolf_core/agents/graph/nodes.py`
- 在 `reasoning_node` 等核心节点中，通过查询 `PlayerRecord` 关联的 `AIPlayerProfile`，动态获取该 AI 玩家配置的 `model_name` 和 `system_prompt`，从而实现不同 AI 玩家的个性化表现。

## 3. 数据流转与状态同步

### 3.1 玩家创建与对局初始化
1. 用户通过 API 发起创建对局请求，包含 AI 玩家配置。
2. `games.py` 解析配置，必要时在 `AIPlayerProfile` 中动态创建新玩家。
3. 初始化 `PlayerRecord`，绑定 `ai_profile_id`。
4. `PlayerStatusManager` 将初始状态写入 Redis 缓存。

### 3.2 对局内状态更新
1. 游戏引擎（如夜晚狼人杀人、白天投票出局）触发玩家状态变更。
2. 调用 `PlayerStatusManager.mark_dead()` 等方法。
3. 更新 Redis 中的 Bitmap/Hash 缓存。
4. 异步提交 UPDATE 语句到 PostgreSQL，保证数据最终一致性。

### 3.3 跨局数据统计
1. 对局结束时，触发 `GAME_OVER_EVENT`。
2. 异步统计任务（Celery Worker）消费事件。
3. 汇总本局玩家的表现（胜负、耗时、异常次数），更新 `AIPlayerStats` 表。

## 4. 扩展性设计
- **模型解耦**: AI 玩家的配置与具体的 LLM 适配器解耦，通过 `model_name` 动态路由到对应的模型服务。
- **缓存降级**: `PlayerStatusManager` 实现了从 Redis 到 DB 的降级查询路径（如 `_is_alive_from_db`），提升系统可用性。
- **自进化预留**: 独立的 `AIPlayerProfile` 和 `AIPlayerStats` 为后续引入强化学习、Prompt 自动微调和经验记忆库提供了数据基础。
