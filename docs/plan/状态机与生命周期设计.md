# 状态机与生命周期设计 (Phase State Machine & Lifecycle)

## 1. 模块职责概述
本模块是 Game Engine 的核心骨架，负责管理整局狼人杀游戏的生命周期（创建、初始化、销毁）以及对局内各个阶段（白天、黑夜、投票等）的严格流转。它必须是**硬编码的绝对裁判**，不允许 LLM 参与状态流转的决策。

## 2. 核心架构设计
模块分为两个主要部分：
- **Lifecycle Manager (生命周期管理器)**：管理全局状态（`INIT -> START -> RUNNING -> SETTLING -> FINISHED`）。
- **Phase State Machine (阶段状态机)**：管理 `RUNNING` 状态下的具体游戏阶段流转（`NIGHT -> DAY_DISCUSSION -> DAY_VOTE -> LAST_WORD -> CHECK_WINNER -> NEXT_ROUND`）。

设计原则：
- **状态不可逆与严格校验**：状态流转必须遵循预定义的有向图，非法流转请求直接拒绝。
- **事件驱动**：状态的进入（`on_enter`）和退出（`on_exit`）必须通过 Event Bus 广播事件，解耦其他模块（如前端推送、Agent 唤醒）。

## 4. 关键数据流转过程
1. **初始化**：API 接收创建房间请求，调用 `LifecycleManager` 初始化，状态设为 `INIT`。
2. **开局**：分配身份完毕，调用 `start_game`，状态变为 `RUNNING`，`PhaseStateMachine` 初始化为 `Round 1, NIGHT`。
3. **阶段流转**：
   - 引擎内部逻辑（如夜晚行动全部提交完毕，或倒计时结束）触发 `transition_to(DAY_DISCUSSION)`。
   - 状态机校验合法性。
   - 状态机调用 `event_bus.publish(Event(type="PHASE_CHANGED", data={"old": "NIGHT", "new": "DAY_DISCUSSION"}))`。
   - 监听该事件的模块（如 Agent Runtime）被唤醒，开始执行白天逻辑。

## 5. 依赖关系说明
- **Event Bus (`ai_werewolf_core/core/event/bus.py`)**：强依赖。状态机不直接调用 Agent 或前端 API，而是通过发布 `PHASE_CHANGED` 等事件来驱动外部系统。
- **Logger (`ai_werewolf_core/utils/logger.py`)**：强依赖。每次状态流转必须记录结构化日志，包含 `game_id`, `old_phase`, `new_phase`, `round`。
- **Data Models (`ai_werewolf_core/db/models.py`)**：状态变更需要持久化到数据库，以便支持断线重连和复盘。

**核心伪代码 (`state_machine.py`)**:
```python
from ai_werewolf_core.schemas.enums import GamePhase
from ai_werewolf_core.core.event.bus import EventBus, Event
from ai_werewolf_core.utils.logger import get_logger

logger = get_logger(__name__)

class PhaseStateMachine:
    VALID_TRANSITIONS = {
        None: [GamePhase.NIGHT],
        GamePhase.NIGHT: [GamePhase.DAY_DISCUSSION],
        GamePhase.DAY_DISCUSSION: [GamePhase.DAY_VOTE],
        GamePhase.DAY_VOTE: [GamePhase.LAST_WORD, GamePhase.CHECK_WINNER],
        GamePhase.LAST_WORD: [GamePhase.CHECK_WINNER],
        GamePhase.CHECK_WINNER: [GamePhase.NEXT_ROUND, None], # None means game over
        GamePhase.NEXT_ROUND: [GamePhase.NIGHT]
    }

    async def transition_to(self, next_phase: GamePhase, context: dict = None):
        if next_phase not in self.VALID_TRANSITIONS.get(self.current_phase, []):
            raise InvalidTransitionError(f"Cannot transition from {self.current_phase} to {next_phase}")
        
        old_phase = self.current_phase
        self.current_phase = next_phase
        
        if next_phase == GamePhase.NEXT_ROUND:
            self.round += 1
            
        logger.info("phase_transition", game_id=self.game_id, old=old_phase, new=next_phase, round=self.round)
        
        await self.event_bus.publish(Event(
            topic="game.phase_changed",
            payload={"game_id": self.game_id, "old_phase": old_phase, "new_phase": next_phase, "round": self.round}
        ))
