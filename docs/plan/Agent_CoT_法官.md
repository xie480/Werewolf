# 法官思考链 (CoT) 模板

## 1. 夜间结算与白天宣布阶段
<thought>
【局势回顾与信息提取】：汇总系统底层传来的所有夜间行动数据，包括狼人击杀目标、女巫用药情况、预言家查验目标及结果等。
【状态更新与规则校验】：根据游戏规则，结算各种技能的叠加效果（如狼人刀中且女巫未救则死亡，女巫救则平安夜，女巫毒则死亡）。更新玩家的存活状态。
【信息披露边界检查】：严格审查即将向全场公布的信息。确保只公布最终的死讯或平安夜，绝对不能泄露任何神职技能的使用细节或查验结果。
【行动策略与风险评估】：制定白天开场的播报策略。评估播报语气是否绝对中立，避免任何可能暗示夜间具体过程的词汇。
【输出内容规划】：规划法官播报台词，宣布天亮、昨夜伤亡情况，并指示下一步流程（如死者遗言或直接进入发言）。
</thought>
[输出法官播报内容，宣布昨夜结果并引导流程]

## 2. 组织发言阶段
<thought>
【局势回顾与信息提取】：确认当前存活玩家列表，确定本轮发言的起始位置和顺序（如死左死右，或警长指定）。
【流程控制与状态监控】：监控当前发言玩家的状态，记录其发言时间，准备在超时或违规时进行干预。
【公平性与秩序维护】：确保发言过程中没有其他玩家插话或进行场外干扰，维护游戏的绝对公平与秩序。
【行动策略与风险评估】：决定何时切断当前玩家的发言并移交给下一位。评估流程推进的节奏是否合理。
【输出内容规划】：规划引导发言的台词，如提示某位玩家开始发言，或宣布发言结束。
</thought>
[输出法官引导台词，控制发言流程]

## 3. 组织投票阶段
<thought>
【局势回顾与信息提取】：确认所有玩家发言完毕，提取系统状态，准备进入放逐投票环节。
【规则校验与权限确认】：确认当前哪些玩家拥有投票权（存活玩家），哪些玩家没有投票权。
【流程控制与状态监控】：下达投票指令，开启倒计时，监控系统接收到的投票数据，确保所有有效票都被记录。
【行动策略与风险评估】：确保投票过程的公开透明，防止任何形式的作弊或延迟投票。
【输出内容规划】：规划宣布投票开始和结束的台词，并准备宣读最终的票型分布。
</thought>
[输出法官台词，宣布投票开始、结束及票型结果]

## 4. 平票PK处理阶段
<thought>
【局势回顾与信息提取】：分析刚刚结束的投票结果，确认出现最高票数相同的平票情况，提取平票玩家名单。
【规则校验与流程重置】：根据平票规则，启动PK流程。确认PK台上的玩家仅有发言权无投票权，其余玩家重新投票。
【流程控制与状态监控】：组织PK玩家依次发言，随后再次组织外围玩家进行投票，监控二次投票结果。
【行动策略与风险评估】：确保PK流程严格按照规则执行，处理可能出现的二次平票等极端边界情况。
【输出内容规划】：规划宣布进入PK阶段的台词，引导PK发言，并宣布最终的PK投票结果。
</thought>
[输出法官台词，引导PK流程并宣布结果]

## 5. 遗言与出局结算阶段
<thought>
【局势回顾与信息提取】：确认刚刚被投票放逐或夜间死亡的玩家，检查游戏规则中关于该阶段是否允许发表遗言的设定（如首夜死有遗言，白天推有遗言）。
【特殊技能触发检查】：检查出局玩家是否拥有出局时可发动的特殊技能（如猎人开枪），如果有，需优先处理技能结算。
【流程控制与状态监控】：引导出局玩家发表遗言，监控遗言内容，结束后宣布其正式离场，并推进游戏进入下一个黑夜。
【行动策略与风险评估】：确保出局结算流程的完整性，防止遗漏特殊技能的触发，维持游戏的连贯性。
【输出内容规划】：规划引导遗言的台词，以及宣布进入黑夜的台词。
</thought>
[输出法官台词，引导遗言并宣布进入黑夜]
