# 角色与能力系统设计 (Role & Ability System)

## 1. 模块职责概述
本模块负责定义狼人杀游戏中所有角色的基础属性、阵营归属以及在不同游戏阶段的行动权限。它是 Game Engine 进行动作校验（Action Validation）的基础，确保“死人不能说话”、“平民不能在夜晚杀人”等基本规则得到严格执行。

## 2. 核心架构设计
采用面向对象的设计，定义一个基础的 `Role` 抽象类，所有具体角色（如 `Werewolf`, `Seer`, `Witch`, `Villager`）继承自该基类。
- **属性定义**：角色名称、所属阵营（狼人/好人）、是否拥有夜间行动能力。
- **权限校验**：提供 `can_act(phase, action_type)` 方法，供引擎在接收到 Agent 动作时进行合法性校验。
- **状态管理**：管理角色在当前对局中的状态（存活/死亡、技能是否已使用，如女巫的解药/毒药）。

## 3. 接口与方法定义

### 3.1 阵营与角色枚举 (`schemas/enums.py` 补充)
```python
class Faction(str, Enum):
    VILLAGER = "VILLAGER"   # 好人阵营
    WEREWOLF = "WEREWOLF"   # 狼人阵营

class RoleType(str, Enum):
    VILLAGER = "VILLAGER"
    WEREWOLF = "WEREWOLF"
    SEER = "SEER"
    WITCH = "WITCH"
    HUNTER = "HUNTER"
```

### 3.2 核心类接口
```python
class BaseRole(ABC):
    role_type: RoleType
    faction: Faction
    
    def __init__(self, player_id: str):
        self.player_id = player_id
        self.is_alive: bool = True
        
    @abstractmethod
    def can_act(self, phase: GamePhase, action_type: ActionType) -> bool:
        """判断该角色在指定阶段是否允许执行指定动作"""
        pass

class WitchRole(BaseRole):
    role_type = RoleType.WITCH
    faction = Faction.VILLAGER
    
    def __init__(self, player_id: str):
        super().__init__(player_id)
        self.has_antidote: bool = True
        self.has_poison: bool = True
        
    def can_act(self, phase: GamePhase, action_type: ActionType) -> bool:
        if not self.is_alive:
            return False
        if phase == GamePhase.NIGHT and action_type in [ActionType.SAVE, ActionType.POISON]:
            return True
        # ... 其他通用动作校验 (如白天投票)
        return False
```

## 4. 关键数据流转过程
1. **角色分配**：在 `INIT` 阶段，引擎根据房间配置实例化对应的 `Role` 对象，并绑定到具体的 `player_id`。
2. **动作校验**：
   - Agent 提交 `Action Schema`（如 `{"action_type": "KILL", "actor_id": "p1", "phase": "NIGHT"}`）。
   - 引擎的 `Action Resolver` 接收到动作，首先查找 `p1` 对应的 `Role` 实例。
   - 调用 `role.can_act(GamePhase.NIGHT, ActionType.KILL)`。
   - 如果返回 `False`，引擎直接拒绝该动作，并可能要求 Agent 重试。
3. **状态更新**：如果动作合法并最终结算成功（如女巫成功使用解药），引擎调用 `role.has_antidote = False` 更新角色内部状态。

## 5. 依赖关系说明
- **Schemas (`ai_werewolf_core/schemas/enums.py`)**：强依赖。角色类型、阵营、动作类型必须使用预定义的枚举。
- **Action Resolver (`ai_werewolf_core/core/engine/resolver.py`)**：被依赖。解析器在处理动作前必须调用本模块进行权限校验。

from abc import ABC, abstractmethod
from ai_werewolf_core.schemas.enums import GamePhase, ActionType, RoleType, Faction

class BaseRole(ABC):
    role_type: RoleType
    faction: Faction
    
    def __init__(self, player_id: str):
        self.player_id = player_id
        self.is_alive: bool = True
        
    def can_perform_common_action(self, phase: GamePhase, action_type: ActionType) -> bool:
        """所有存活玩家的通用动作校验"""
        if not self.is_alive:
            return False
        if phase == GamePhase.DAY_DISCUSSION and action_type == ActionType.SPEAK:
            return True
        if phase == GamePhase.DAY_VOTE and action_type == ActionType.VOTE:
            return True
        return False

    @abstractmethod
    def can_act(self, phase: GamePhase, action_type: ActionType) -> bool:
        pass
