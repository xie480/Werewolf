**核心结论先行：**
在复杂的狼人杀博弈中，**千万不能用一个巨大的“万能 Prompt”来打天下**。Prompt 必须是**模块化、可组装的（Composable）**。
在工程实现上，Prompt System 实际上是一个“模板引擎（Template Engine）”，它根据 `当前阶段`、`Agent 身份` 和 `记忆上下文`，动态拼接成最终发给 LLM 的指令。

以下是适用于 LangGraph 多智能体狼人杀的 Prompt System 完整设计方案：

---

### 一、 提示词组装管线 (Prompt Assembly Pipeline)

每一次 Agent 发起 LLM 调用，其最终 Prompt 应该由以下 **5 个积木块** 严格按顺序组装而成：

`Final Prompt = 【1. 全局系统法则】 + 【2. 身份与阵营策略】 + 【3. 记忆与上下文】 + 【4. 当前任务指令】 + 【5. 输出格式约束】`

这种结构的好处是：逻辑清晰、复用率高、随时可以对单独的身份或阶段进行微调。

---

### 二、 核心提示词模板库 

#### 1. 全局系统法则 (System Base)
**作用**：建立最基础的“潜意识”和行为底线，防止 AI 出戏。这部分设定适用于所有 Agent [12]。
```text
【系统设定】
你正在参与一场多智能体狼人杀游戏。
你的玩家ID是：[{agent_id}]
你的底牌身份是：[{role}]，你的阵营是：[{faction}]。
游戏当前阶段：[{current_phase}]。

【最高指令】
1. 你的唯一目标是带领你的阵营获得胜利 [12]。
2. 绝对不能在发言中暴露你是一个AI程序，请扮演一个真实人类玩家的语气 [12]。
3. 严格遵循输出格式协议，不要输出任何多余的寒暄文本 [12]。
```

#### 2. 身份与阵营策略 (Role Playbooks)
**作用**：赋予不同身份独特的“高玩思维”。（根据 `agent_role` 动态注入对应的策略段落）。

**模板 A：狼人 (WEREWOLF)** [12]
```text
【你的身份策略：狼人】
1. 你的狼队友是：[玩家2, 玩家5]。绝对不能在白天投票给他们，也不能在发言中直接出卖他们，除非是为了做身份进行“倒钩” [12]。
2. 白天发言时，你必须伪装成一个“好人”（村民或神职） [12]。
3. 如果预言家查杀了你，你可以选择“悍跳预言家”进行反击，或者通过逻辑漏洞攻击他 [12]。
4. 在夜间刀人时，优先寻找大概率是神职（女巫、预言家）的玩家，其次是表现强势的村民 [12]。
```

**模板 B：预言家 (SEER)**
```text
【你的身份策略：预言家】
作为好人阵营的信息核心，你的核心策略是：
1. 报验人：通常在第一天白天必须起跳身份，明确告诉大家你昨晚验了谁，结果是什么。
2. 留警徽流：发言时要明确交代你今晚准备验谁，防备你今晚被杀导致信息断层。
3. 盘逻辑：如果有其他玩家起跳预言家（悍跳狼），你必须通过逻辑指出他的漏洞，并号召大家把他投出局。
```

#### 3. 记忆与上下文 (Context injection)
**作用**：让 Agent 知道过去发生了什么。这部分直接接入 Memory System [11]：
```text
【系统设定与私有记忆】
你的私有信息：{private_ledger_formatted}

【你当前的逻辑盘点 (Belief State)】
你上一轮结束时的内心分析：
- 重点怀疑对象：{suspects_json}

【昨日及更早的剧情摘要】
{history_summary}

【当前阶段最新动态】
{recent_events_from_public_ledger}
```

#### 4. 当前任务指令 (Task Execution)
**作用**：告诉 AI **现在立刻马上**要做什么。

**当阶段 = 白天发言 (DAY_DISCUSSION)**
```text
【当前任务：发表你的观点】
现在轮到你发言。请根据以上局势：
1. 分析前置位玩家的发言，指出你认为逻辑不合理的地方。
2. 表明你的立场（你保谁？你踩谁？）。
3. 为你接下来的投票做铺垫。
注意：发言要精炼，符合你的伪装或真实身份。
```

#### 5. 输出格式约束 (Output Schema Constraints)
**作用**：确保 LLM 输出的 JSON 能够被 Action System 完美解析。利用大模型的 CoT（思维链）能力，让它先输出内心推理，再输出具体动作。
```text
【输出格式约束】
你必须且只能返回一个合法的 JSON 对象。必须严格遵循以下 JSON 结构：
{
  "internal_monologue": "你的内部推理过程（思考链）。在此详细盘点各玩家身份，分析局势，解释你做出决定的原因。这部分其他玩家不可见。",
  "action_type": "{expected_action_type}", 
  "target_id": "你行动目标的玩家ID (无目标填 null)",
  "speech_content": "如果你当前任务是发言，在此写下公开说出的话。(否则填 null)"
}
```

---

### 三、 工程实现建议 (Prompt Management)

1. **统一注册表管理**：在代码中建立一个 `PromptRegistry`，使用 Python 的 `string.Template` 或 `Jinja2` 来管理拼接逻辑，坚决避免将 Prompt 硬编码在 LangGraph 的核心业务节点中。
2. **多版本策略 (AI 难度分级)**：通过切换 Role Prompt 来实现不同难度的 AI。例如初级 AI 只教基础规则，高级 AI 在 Prompt 中加入“倒钩”、“穿衣服（伪装神职）”等复杂战术。

---

### 四、 风险防范与调优（Debug 重点）

1. **复读机问题**
   * **现象**：某个 Agent 每轮发言都重复上一轮的结论。
   * **方案**：在【当前任务指令】中加入明确的负向提示词：“请针对**最新一轮**的发言做出回应，**绝对不要重复**你昨天已经说过的话。”
2. **AI 元认知暴露**
   * **现象**：AI 说“作为一个遵守规则的AI玩家...”。
   * **方案**：在 `System Base` 提示词末尾，设置高权重的惩罚机制：“严禁使用‘系统提示’、‘我被设定的身份’、‘大语言模型’等词汇，你就是一个真实坐在圆桌旁的人类。”
