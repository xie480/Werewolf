**核心结论先行：**
在 AI 多智能体评测中，**绝对不能仅仅以“胜率”作为唯一指标**。狼人杀具有极大的随机性，单纯看胜负无法体现 AI 的“智商”。
我们要构建一套 **“过程导向 + 结果导向”相结合的量化评测体系**。依托上一节持久化下来的 `game_events`（事件流水）和 `agent_belief_states`（内部认知），采用 **LLM-as-a-Judge（大模型作为裁判）** 与 **启发式脚本（Heuristic Rules）** 双轨并行的评分机制。
以下是专为 AI 狼人杀设计的多维度 Evaluation System 落地方案：
---
### 一、 核心评测维度设计 (五维雷达图)
建议为每个 Agent 在每局游戏后生成一个五维雷达图数据，具体维度及计算方式如下：
#### 1. 规则服从度 (Rule Compliance) - 脚本计算 (客观)
* **定义**：AI 是否严格遵守了 JSON 格式化输出和游戏规则。
* **计算公式**`100 - (格式错误重试次数 + 非法动作拦截次数) * 10`。
* **意义**：评估基础指令遵循能力。
#### 2. 逻辑连贯性 (Logical Consistency) - 脚本+内部状态对比 (客观+主观)
* **定义**：AI 白天的发言、投票行为与其内心的 `Belief State`（信任热力图）是否一致。
* **举例**：如果 AI 内部信念判定 3 号是铁狼，但发言保 3 号，且自身不是在做“倒钩”战术，则扣分。
#### 3. 伪装与欺骗能力 (Deception) - LLM 裁判打分 (主观)
* **定义**：狼人专属指标。评估狼人发言是否像好人，能否成功隐藏身份。
* **计算依据**：好人玩家的内部状态中，对该狼人的“怀疑度”平均值。怀疑度越低，欺骗得分越高。
#### 4. 煽动与说服力 (Persuasion/Leadership) - 数据统计 (客观)
* **定义**：评估该 Agent 提出的观点被其他玩家采纳的程度。
* **计算依据**：该 Agent 在发言中明确号召票出 X 号，后续投票环节中，有多少其他玩家跟随他把票投给了 X 号。跟票率越高，得分越高。
#### 5. 态势感知与推理 (Situational Awareness) - 脚本对比 (客观)
* **定义**：好人专属指标。评估 AI “找狼”的准确率。
* **计算依据**：对比 Agent 每轮结束时的 `suspect_heatmap`（嫌疑人名单）与上帝视角的“真实狼人名单”的重合度（IoU 或精准率/召回率）。
---
### 二、 评测管线架构 (Evaluation Pipeline)
评测不应该在游戏对局中实时进行（会严重拖慢游戏速度且浪费 Token），而是**在 `GAME_OVER` 阶段触发异步离线计算**。
1. **Data Extraction (数据抽取)**：游戏结束后，评测引擎从 `game_events` 表拉取本局完整时间线，从 `agent_belief_states` 拉取全员心路历程。
2. **Rule-based Scoring (规则打分)**：通过 Python 脚本计算“格式重试率”、“跟票率”、“找狼准确率”等客观数据。
3. **LLM-as-a-Judge (大模型裁判)**：将关键对峙回合（如预言家与悍跳狼的对线）打包成 Prompt，发给拥有上帝视角的最强模型（如 GPT-4o 或 Claude-3.5-Sonnet）进行质性分析与打分。
4. **Report Generation (报告生成)**：汇总生成该局的全局复盘日志（JSON 格式），落库入 `match_reports` 表。
---
### 三、 LLM-as-a-Judge 核心 Prompt 模板
你需要一个独立的、不参与游戏的“法官模型”来评判玩家的表现。可以直接使用以下 Prompt 模板：
```text
【裁判任务说明】
你是一个专业的狼人杀多智能体系统评测裁判。
现在有一局游戏已经结束，请你根据以下提供的【上帝视角数据】和【该玩家的全部发言与内部思维】，对玩家 [{agent_id}] 进行行为评分与复盘分析。
【全局真实身份数据】
{global_roles_json}
【被评测玩家数据】
该玩家底牌：{agent_role}
该玩家所属阵营：{agent_faction}
【该玩家的关键行为日志 (按时间线)】
{agent_action_and_speech_logs}
【该玩家的内部思维链 (Belief State)】
{agent_internal_monologues}
【评测任务约束】
请你根据以上信息，严格输出一个 JSON 格式的评测报告。不要输出其他废话。
JSON 结构如下：
{
  "deception_score": 0-10, // 伪装度（如果是好人，填 null）
  "reasoning_score": 0-10, // 逻辑推理得分（满分10，基于他是否找到了真相，或作为狼人是否编造了完美逻辑）
  "roleplay_score": 0-10, // 角色扮演得分（满分10，是否有出戏表现或AI特有的话术痕迹）
  "strengths": "简短的一句话总结他的高光时刻/优点",
  "weaknesses": "简短的一句话总结他的致命失误/逻辑漏洞",
  "overall_review": "一段100字左右的综合评价，用于展示在前端复盘大屏上。"
}
```

### 五、 评测系统的避坑指南
1. **防止“结果倒推过程”的偏见 (Hindsight Bias)**
   * **问题**：LLM 裁判知道了全局真相后，如果发现好人玩家在第一天没找出狼，很容易给出低分。但这在游戏初期是完全正常的。
   * **解决方案**：在 LLM-as-a-Judge 的 Prompt 中加入强调指令`“请剥离上帝视角，仅根据该玩家当时能够获取的公开信息，评判其推理是否合理。哪怕他的结论是错的，只要当时逻辑自洽，就不应扣减逻辑推理分。”`
2. **打分尺度漂移**
   * **问题**：大模型打分容易偏向中庸（总是给 7 分或 8 分），或者随着时间推移尺度不一。
   * **解决方案**：在 Prompt 中给出明确的 **Rubric（评分尺规）**。例如补充说明：“9-10分代表完美骗过所有人/找出所有狼；6-8分代表表现正常但无亮眼发挥；3-5分代表出现明显逻辑漏洞但未暴露；0-2分代表自爆或违规”。
3. **大批量并发调用的成本评估**
   * 如果 12 个玩家每人一份 Evaluation，一局游戏结束后要并发调 12 次高智商大模型（如 GPT-4），可能产生 1-2 美元的纯复盘成本。
   * **建议**：复盘可以使用价格更便宜的模型（如 `deepseek-chat` 或 `gpt-4o-mini`），因为做信息总结和评分的任务难度低于直接做多智能体博弈决策。
